    [!WARNING] This page in the Chef Open Source Software Practices repository is currently undergoing content review.

    Project Name

    Chef InSpec — A framework for testing and auditing your infrastructure to ensure compliance with security and policy requirements.

    Project Scope

    • Chef InSpec allows users to write tests in a human-readable language to verify the state of systems.
    • It solves the problem of ensuring that infrastructure complies with security policies, organizational rules, and regulatory standards.
    • Built in Ruby, relies on a flexible plugin system, and supports integration with cloud APIs, OS-level inspection, and configuration management tools.

    Communication

    • Slack Channel: #inspec
    • Triage Schedule: n/a
    • Release Notes: Docs Discourse
    • Discourse topics: InSpec @ Chef Discourse

    Sub Projects

    • Train — A transport layer and backend abstraction used by InSpec to connect and inspect various targets.
    • Resource Packs — Reusable collections of custom resources and plugins shared across compliance profiles.
    • Profiles - Reusable collections compliance profiles.
